1. CSV Module: The csv module in Python is a built-in library used to read, write, and manipulate CSV (Comma-Separated Values) files.

A CSV file stores tabular data (like spreadsheets or databases) in plain text format.

Example CSV file:

name,age,country
Nasif,25,Canada
John,30,USA

2. Purpose:

The csv module is used to:

| Purpose               | Explanation                   |
| --------------------- | ----------------------------- |
| Read CSV files        | Extract data row by row       |
| Write CSV files       | Save data into CSV format     |
| Process tabular data  | Work with rows and columns    |
| Handle large datasets | Efficient file processing     |
| Exchange data         | Common format between systems |

3. Key Concepts

| Term      | Meaning                                      |
| --------- | -------------------------------------------- |
| Row       | A single line in CSV                         |
| Column    | A field in a row                             |
| Delimiter | Character separating values (`,` by default) |
| Header    | First row (column names)                     |

4. Reading CSV Files

Basic Reading (csv.reader)

import csv

with open("data.csv", "r") as file:
    reader = csv.reader(file)
    for row in reader:
        print(row)

Output:

['name', 'age', 'country']
['Nasif', '25', 'Canada']
['John', '30', 'USA']

Key Points:

- Each row is returned as a list

- All values are read as strings

- Use a loop to iterate rows

5. Writing CSV Files

Basic Writing (csv.writer)

import csv

with open("output.csv", "w", newline="") as file:
    writer = csv.writer(file)
    writer.writerow(["name", "age", "country"])
    writer.writerow(["Nasif", 25, "Canada"])

Key Points:

- writerow() writes one row

- writerows() writes multiple rows

- newline="" prevents blank lines (important on Windows)

6. Reading CSV as Dictionary

Using csv.DictReader

import csv

with open("data.csv", "r") as file:
    reader = csv.DictReader(file)
    for row in reader:
        print(row)

Output:

{'name': 'Nasif', 'age': '25', 'country': 'Canada'}

Key Points:

- Each row is a dictionary

- Keys come from the header row

- Easier to work with than lists

7. Writing CSV as Dictionary

Using csv.DictWriter

import csv

with open("output.csv", "w", newline="") as file:
    fieldnames = ["name", "age", "country"]
    
    writer = csv.DictWriter(file, fieldnames=fieldnames)
    
    writer.writeheader()
    writer.writerow({"name": "Nasif", "age": 25, "country": "Canada"})

Key Points:

- Must define fieldnames

- writeheader() writes column names

- Data must match keys

8. Custom Delimiters

CSV doesn't always use commas.

Example with Semicolon ;

import csv

with open("data.csv", "r") as file:
    reader = csv.reader(file, delimiter=";")
    for row in reader:
        print(row)

9. Handling Quotes

CSV files may include quoted text:

name,comment
Nasif,"Hello, world!"

Python handles this automatically:

reader = csv.reader(file)

10. Skipping Header Row

import csv

with open("data.csv", "r") as file:
    reader = csv.reader(file)
    next(reader)  # Skip header
    
    for row in reader:
        print(row)

11. Writing Multiple Rows

rows = [
    ["Nasif", 25, "Canada"],
    ["John", 30, "USA"]
]

with open("output.csv", "w", newline="") as file:
    writer = csv.writer(file)
    writer.writerows(rows)

12. File Modes

| Mode  | Meaning           |
| ----- | ----------------- |
| `"r"` | Read              |
| `"w"` | Write (overwrite) |
| `"a"` | Append            |

13. Common Parameters

| Parameter   | Description                 |
| ----------- | --------------------------- |
| `delimiter` | Character separating values |
| `quotechar` | Character used for quoting  |
| `newline`   | Prevents extra blank lines  |
| `encoding`  | File encoding (e.g., UTF-8) |

14. Real-World Example

Scenario: Filter Users by Country

import csv

with open("data.csv", "r") as file:
    reader = csv.DictReader(file)
    
    for row in reader:
        if row["country"] == "Canada":
            print(row["name"])

15. Limitations

- All data is read as strings

- No automatic type conversion

- Large files may require optimization

- Not suitable for complex data (use JSON instead)

16. Common Mistakes

❌ Forgetting newline=""

open("file.csv", "w")  # May cause blank lines

❌ Not handling encoding

open("file.csv", "r", encoding="utf-8")

❌ Assuming numbers are integers

age = int(row["age"])  # Must convert manually

17. Comparison Table

| Feature        | `csv.reader`     | `csv.DictReader`    |
| -------------- | ---------------- | ------------------- |
| Output         | List             | Dictionary          |
| Access         | Index (`row[0]`) | Key (`row["name"]`) |
| Ease of Use    | Moderate         | Easy                |
| Header Support | Manual           | Automatic           |

18. Key Points for Interviews:

- csv is a built-in Python module

- Used for reading/writing tabular data

- reader → list output

- DictReader → dictionary output

- Always handle newline properly

- Data is always read as strings

- Supports custom delimiters
