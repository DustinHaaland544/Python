Python Study Notes

Topic: *args and **kwargs

1. Definition

*args: A special parameter that allows a function to accept any number of positional arguments.
The arguments are stored inside a tuple.

**kwargs: A special parameter that allows a function to accept any number of keyword arguments.
The arguments are stored inside a dictionary.

2. Purpose

| Feature       | Purpose                                                                     |
| ------------- | --------------------------------------------------------------------------- |
| `*args`       | Makes functions flexible when the number of positional arguments is unknown |
| `**kwargs`    | Allows flexible keyword-based configuration                                 |
| Both together | Useful in scalable applications, APIs, decorators, and frameworks           |

3. Syntax

Basic Syntax

def function_name(*args):
    pass

def function_name(**kwargs):
    pass

Combined Syntax (Correct Order):

def function_name(normal_parameters, *args, **kwargs):
    pass

Parameter Order Rule ⚠️

1. Normal parameters

2. *args

3. **kwargs

Incorrect order will raise an error.

4. *args – Detailed Explanation

Storage Type

- Stored as a tuple

- Tuple is immutable

Example 1: Basic Usage

def show_numbers(*args):
    print(args)

show_numbers(1, 2, 3, 4)

Output:

(1, 2, 3, 4)

Example 2: Practical Usage

def add_numbers(*args):
    total = 0
    for num in args:
        total += num
    return total

print(add_numbers(10, 20, 30))

Output:

60

Key Points

- * packs multiple positional arguments into a tuple.

- The name args is just a convention (can be changed).

- Useful when argument count is unknown.

5. **kwargs – Detailed Explanation

Storage Type

- Stored as a dictionary

- Key-value pairs

Example 1: Basic Usage

def show_details(**kwargs):
    print(kwargs)

show_details(name="Nasif", age=25)

Output:

{'name': 'Nasif', 'age': 25}

Example 2: Looping Through Keyword Arguments

def show_details(**kwargs):
    for key, value in kwargs.items():
        print(key, "=", value)

show_details(name="Nasif", country="Canada")

Output:

name = Nasif
country = Canada

Key Points

- ** packs keyword arguments into a dictionary.

- Keys become parameter names.

- Very useful for configurable functions.

6. Using *args and **kwargs Together

def demo(*args, **kwargs):
    print("Positional:", args)
    print("Keyword:", kwargs)

demo(1, 2, 3, name="Nasif", age=25)

Output:

Positional: (1, 2, 3)
Keyword: {'name': 'Nasif', 'age': 25}

7. Argument Unpacking (Very Important Concept)

Unpacking List Using *

def add(a, b, c):
    return a + b + c

numbers = [1, 2, 3]
print(add(*numbers))

Equivalent to:

add(1, 2, 3)

Unpacking Dictionary Using **

def show(name, age):
    print(name, age)

data = {"name": "Nasif", "age": 25}
show(**data)

Equivalent to:

show(name="Nasif", age=25)

8. Real-World Example

def create_user(username, *roles, **details):
    print("Username:", username)
    print("Roles:", roles)
    print("Details:", details)

create_user(
    "Nasif",
    "admin",
    "editor",
    email="nasif@email.com",
    country="Canada"
)

Output:

Username: Nasif
Roles: ('admin', 'editor')
Details: {'email': 'nasif@email.com', 'country': 'Canada'}

9. Common Mistakes

Forgetting the stars

def func(args):   # Not *args

Wrong parameter order

def func(**kwargs, *args):   # Invalid

Confusing positional vs keyword arguments

func(10)              # Positional
func(number=10)       # Keyword

func(10)              # Positional
func(number=10)       # Keyword

10. Advanced Concept – Argument Forwarding

Common in decorators and wrapper functions:

def wrapper(*args, **kwargs):
    return original_function(*args, **kwargs)

11. Comparison Table

| Feature     | `*args`                  | `**kwargs`                   |
| ----------- | ------------------------ | ---------------------------- |
| Accepts     | Positional arguments     | Keyword arguments            |
| Stored As   | Tuple                    | Dictionary                   |
| Symbol      | `*`                      | `**`                         |
| When to Use | Unknown number of values | Flexible named configuration |
| Can Unpack? | Yes (`*list`)            | Yes (`**dict`)               |

12. Key Exam / Interview Points:

- *args collects extra positional arguments into a tuple.

- **kwargs collects extra keyword arguments into a dictionary.

- Order must always be: normal → *args → **kwargs.

- Can be used in both function definition and function call.

- Frequently used in decorators and large-scale applications.
